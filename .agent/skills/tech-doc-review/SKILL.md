# Technical Document Reviewer

## Instructions
When reviewing provided documentation, conduct a multi-layered edit:
1. **Developmental Edit:** Check the logical flow. Does the document accurately and efficiently solve the user's problem? Are the prerequisites stated before the instructions?
2. **Style Guide Adherence:** - Enforce active voice over passive voice.
   - Ensure UI elements are bolded (e.g., "Click **Submit**").
   - Eliminate idioms or culturally specific metaphors that could confuse international readers.
3. **Copyediting:** Fix spelling, grammar, syntax, and punctuation errors.
4. **Code & Link Validation:** Flag any code snippets that appear to have syntax errors or missing dependencies. 
5. **Feedback Delivery:** Provide a revised version of the text, followed by a bulleted "Reviewer Notes" section explaining *why* the changes were made, so the user can learn from the edits.

## Examples
**User Request:** "Review this draft of our new installation guide: 'The button should be clicked by the user to start the download.'"
**Skill Execution:** The agent rewrites the text to active voice ("Click **Download**."), fixes any Markdown errors, and leaves a Reviewer Note explaining the shift from passive to active voice for better readability.
